# 数据隐私与提交边界

## 禁止提交

以下内容不得提交到本仓库：

- 任何真实券商结单 PDF、截图、OCR 结果或 raw extraction JSON
- 任何 SQLite 数据库，包括 `investment.sqlite`、`futu_raw_fact.sqlite`、staging DB
- 任何 Excel / CSV / NDJSON / cache / exports 产物
- 真实账户号、客户姓名、文件名中的账号、银行信息、身份证件信息
- 真实交易金额、持仓数量、收益金额、融资利息金额、股息金额
- 能推断个人投资组合的真实标的 + 数量 + 金额组合
- 本机绝对路径、下载目录路径、个人用户名路径

## 可以提交

以下内容可以提交：

- 脱敏后的需求、设计、决策和字段定义
- SQL schema
- 不带个人数据的计算规则和工具代码
- 示例表结构和 `<private>` 占位数据
- 前端页面结构、状态设计和交互说明

## 本地验证建议

真实数据只应放在本地私有目录或私有数据库中。开发前端或计算逻辑时，可以读取本地私有 SQLite 进行验证，但输出到仓库的 snapshot、fixture、测试数据必须脱敏或合成。

## Git Ignore 原则

`.gitignore` 已默认排除常见数据产物。如果新增工具会生成数据文件，必须先更新 `.gitignore`，再运行工具。
