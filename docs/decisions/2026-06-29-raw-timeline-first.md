# 决策：第一阶段优先还原原始事实时间轴

日期：2026-06-29

## 决策

第一阶段优先目标从“尽快定义所有费用/税费和 treatment item”调整为：

> 先把结单里的所有原始数值、事件、现金变化、资产变化和交易行为，按统一框架准确还原成完整时间轴。

费用、税费、成本化、费用化、税务处理、个人交易决策口径等内容，先视为后续运算策略或标记层。它们不应反过来影响原始数据抽取和入库。

## 原因

- 用户确认：只要原始数据严丝合缝，费用和税费 items 可以基于原始事实表手动或自动计算后再标记。
- 原始数据抽取准确性是后续所有收益计算、税务准备和对账的前提。
- 过早把费用/税费 item 当作 P0，会把解析目标和运算策略混在一起，增加误分类和重复统计风险。
- 对真实结单而言，很多费用/税费可能以明细、汇总、净额、隐含差额或备注形式出现；先保留原始事实和来源证据，比先判断 treatment 更稳。

## 影响

- `fee_tax_items`、`treatment_profiles`、`treatment_assignments` 仍保留在模型中，但优先级降为“基于原始事实的后处理/标记/计算层”。
- 下一阶段推进顺序应优先补齐真实结单时间轴：
  1. 股票/期权交易 raw lines 结构化解析。
  2. 现金流水、资产变动、基金订单、IPO、公司行动等已抽取事实的去重和来源校验。
  3. 股票收益计划、融资利息、期权行权/到期等剩余事实模块。
  4. 结单报告余额和持仓快照对账。
  5. 在原始事实时间轴稳定后，再统一生成费用/税费 items、treatment 和计算视图。
- Base 镜像第一阶段应优先展示原始事实时间轴和来源证据，而不是优先展示税务/费用处理结论。

## 非决策

- 本决策不取消 `fee_tax_items` 或 treatment 层。
- 本决策不决定最终税务处理方式。
- 本决策不要求第一阶段手工算完所有收益；它只要求底层事实足够完整、可追溯、可复算。
